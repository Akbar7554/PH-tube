• Discuss the scope of var, let, and const

    • 1. var:
    Function Scope: Variables that are declared with var are function-scoped. This indicates that they are only used within the defined function.

    Hoisting: Var variables are elevated to the top of the function or global scope in which they are used. This means that even if you can access them before they are declared in the code, they will start out with an undefined value.

    No Block Scope:If you define a var variable inside of a block (such as an if statement or a for loop), it is accessible outside of that block since var does not have block-level scope.

    2. let:
    Block Scope:Let-declared variables are block-scoped, which means that only the block in which they are defined can access them. This applies to code blocks like if statements, for loops, and others.
    Hoisting:Although let variables are raised, they are not initialised. You will get a ReferenceError if you attempt to access a let variable before it has been defined in the programme.

    3. const:
    Block Scope: Like let, variables declared with const are block-scoped and only accessible within the block in which they are defined.
    Immutability: Const variables cannot be reassigned after they are declared. However, for objects and arrays declared with const, their properties or elements can still be modified.
    Hoisting: Const variables are also hoisted but not initialized. Similar to let, trying to access a const variable before it's declared will result in a ReferenceError.

    The scope of var is function-scoped, and it lacks block-level scoping, which can lead to unexpected behavior. let and const both have block-level scope, but const is used when you want to declare a variable that should not be reassigned after its initial assignment. It's important to choose the appropriate variable declaration based on your specific use case to avoid scope-related issues in your JavaScript code.

  • Tell us the use cases of null and undefined

    • Null is an object in JavaScript and represents primitive data types. A null value in JavaScript is used for referring absence of any object value and if any function or variable returns null, then we can infer that the object could not be created.
    The undefined property indicates that a variable has not been assigned a value, or not declared at all.

  • What do you mean by REST API?

    • REST is an alternative to methods such as SOAP (Simple Object Access Protocol) and WSDL (Web Services Description Language). It was developed towards the end of the 1990s and fundamentally changed the API landscape. The first companies to use a REST API were eBay and Amazon. Only a selection of partners got access to eBay’s well documented and user-friendly REST API. As a result, eBay’s marketplace was not only accessible through direct visits but through any website that accessed the eBay API. Flickr also launched a REST API in 2004, just in time for the rise of social networking and blogs on the web. This paved the way for social sharing, which Facebook and Twitter later joined. In 2006, Amazon’s REST API helped contribute to the development of the cloud, which is now widely used. RESTful APIs allow users to connect to the cloud and interact with cloud services. They are now accessed by many websites and are considered the backbone of the web.
